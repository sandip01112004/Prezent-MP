PreZent: Smart Bluetooth Attendance System

A reliable, automated system leveraging ESP32 Bluetooth Low Energy (BLE) scanning, Node.js, and MongoDB to simplify student presence detection and management.

View Code Access Portal Demo

Project Overview: What is PreZent?

Problem Statement

Traditional attendance systems are time-consuming and prone to human error or proxy attendance. An automated, proximity-based solution is needed for efficient and accurate class attendance logging.

Objectives

  • Develop a **Bluetooth-based presence detection system** using ESP32.
  • Design a secure **web portal (MERN Stack)** for faculty and students.
  • Ensure **accurate, real-time logging** of presence data to MongoDB.
  • Provide a mechanism for students to **register their unique Bluetooth MAC address**.

Workflow (The Smart Loop)

  1. **Setup:** Faculty starts a lecture on the portal. The ESP32 is triggered.
  2. **Scan:** The **ESP32** scans for all nearby Bluetooth Low Energy (BLE) devices.
  3. **Transmit:** The ESP32 sends the list of scanned **MAC addresses** and signal strength to the Node.js Backend via Wi-Fi.
  4. **Verify:** The Node.js Backend validates the MAC addresses against the **hashed MACs** stored in the MongoDB database.
  5. **Log:** Attendance is marked for recognized students in real-time.

Key Features

  • **Automated Presence:** Eliminates manual roll call, saving time.
  • **Secure Registration:** Students register a single, unique MAC (hashed for privacy).
  • **Role-Based Access:** Separate login/dashboards for Students and Teachers.
  • **Full-Stack Solution:** Includes hardware interface, API, and web interface.
  • **Uniqueness Check:** Prevents duplicate PRN and email registration.

System Architecture: MERN Stack & ESP32

Frontend (Client)

Built with plain **HTML, CSS, and JavaScript**. Provides an intuitive portal for student signup, login, and viewing the **Attendance Dashboard**.

Backend (API & Logic)

A **Node.js/Express** server exposing RESTful APIs. It handles user authentication, data validation, and manages connections to the MongoDB database. It is the crucial link between the hardware and the data layer.

Database (Data Layer)

Uses **MongoDB** (NoSQL) to store student credentials (hashed MAC, PRN, details) and lecture data, offering flexibility and scalability.

Hardware Interface

An **ESP32 microcontroller** acts as the Bluetooth scanner, capturing BLE advertisements from student devices and communicating the raw data to the backend via a dedicated POST API endpoint.
